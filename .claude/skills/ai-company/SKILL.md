# AI Company v3 - Reference-Driven Multi-Agent System

CEO(사용자)가 목표를 입력하고 레퍼런스(참고 자료)를 제공하면, 에이전트들이 협업하여 목표를 달성하는 시스템입니다.

## 워크플로우 (v3)

```
┌─────────────────────────────────────────────────────────────────┐
│                        Phase 1: 계획 수립                        │
├─────────────────────────────────────────────────────────────────┤
│  ① CEO 목표 입력                                                │
│       ↓                                                         │
│  ② RM: 백로그 생성 (프로젝트/태스크)                            │
│       ↓                                                         │
│  ③ CEO: 레퍼런스 첨부 (URL, 이미지, 파일, 메모)                 │
│       ↓                                                         │
│  ④ Tool Agent: 레퍼런스 분석 → Tool 추천 + 검증                 │
│       ↓                                                         │
│  ⑤ CEO: Tool 추천 승인/수정                                     │
│       ↓                                                         │
│  ⑥ HR: 에이전트 고용 + Tool/레퍼런스 할당                       │
│       ↓                                                         │
│  ⑦ RM: 에이전트에 태스크 할당                                   │
├─────────────────────────────────────────────────────────────────┤
│                        Phase 2: 실행                            │
├─────────────────────────────────────────────────────────────────┤
│  ⑧ CEO: 실행할 프로젝트 선택                                    │
│       ↓                                                         │
│  ⑨ Expert: 레퍼런스 참조하며 태스크 수행                        │
│       ↓                                                         │
│  ⑩ 결과 보고                                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 변경사항 (v2 → v3)

| 항목 | v2 | v3 |
|------|----|----|
| 패러다임 | Tool-Driven | **Reference-Driven** |
| CEO 역할 | Tool을 직접 선택 | **레퍼런스 제공 + 추천 승인** |
| Tool Agent | 검증자 (haiku) | **컨설턴트 (sonnet)** |
| Expert 입력 | 태스크 + Tool만 | 태스크 + Tool + **레퍼런스** |
| 결과물 품질 | Expert 추측 의존 | **레퍼런스 기반** 일관된 품질 |

## 실행 방법

### 명령어
- `/ai-company` - 현재 상태 확인 및 계속 진행
- `/ai-company init` - 새 세션 시작
- `/ai-company status` - 현재 상태 요약
- `/ai-company reset` - 모든 상태 초기화

## Step-by-Step 가이드

### Step 1: CEO 목표 입력

사용자에게 목표를 입력받습니다:

```
══════════════════════════════════════════════════════════════
   AI Company v3 - Reference-Driven Multi-Agent System
══════════════════════════════════════════════════════════════

안녕하세요, CEO님! 회사의 목표를 입력해주세요.

예시:
- "쿠팡에 입점하고 월 매출 1000만원 달성"
- "브랜드 인스타그램 운영하고 팔로워 1만명 달성"
- "자동화된 고객 응대 시스템 구축"

목표:
```

목표를 받으면 JSON으로 저장:

```bash
# company/state/session.json
{
  "session_id": "ses_20260206_001",
  "phase": "goal_input",
  "created_at": "2026-02-06T10:00:00Z"
}

# company/state/ceo_goal.json
{
  "goal": "입력받은 목표",
  "created_at": "2026-02-06T10:00:00Z"
}
```

### Step 2: RM 백로그 생성

RM 역할로 목표를 분석하여 백로그를 생성합니다.

**분석 시 고려사항:**
1. 목표 달성에 필요한 프로젝트 식별
2. 각 프로젝트를 구체적 태스크로 분해
3. 태스크 간 의존성 파악
4. 각 태스크에 필요한 Tool 카테고리 제안

**출력 예시:**
```
══════════════════════════════════════════════════════════════
   RM: 백로그 생성 완료
══════════════════════════════════════════════════════════════

프로젝트 1: 쿠팡 입점 준비
   ├─ Task 1.1: 시장 트렌드 조사
   │      제안 Tool: [web_search]
   │
   ├─ Task 1.2: 경쟁사 분석
   │      제안 Tool: [web_search, spreadsheet]
   │
   └─ Task 1.3: 제품 카테고리 선정
          제안 Tool: [analysis]

프로젝트 3: 상품 등록
   ├─ Task 3.1: 제품 상세 페이지 디자인
   │      제안 Tool: [design]
   │
   └─ Task 3.2: 쿠팡 상품 등록
          제안 Tool: [ecommerce_api]
```

저장: `company/state/backlog.json`

### Step 3: CEO 레퍼런스 첨부

CEO에게 각 태스크에 참고 자료를 첨부받습니다.

```
══════════════════════════════════════════════════════════════
   CEO: 레퍼런스 첨부
══════════════════════════════════════════════════════════════

각 태스크에 참고할 자료가 있으면 첨부해주세요.
URL, 이미지, 파일, 또는 텍스트 메모를 입력하실 수 있습니다.
없으면 건너뛰어도 됩니다. Tool Agent가 태스크 유형 기반으로 추천합니다.

─────────────────────────────────────────────────────────────
프로젝트 1: 쿠팡 입점 준비

  Task 1.1: 시장 트렌드 조사
     레퍼런스: (없으면 Enter)
     > https://datalab.naver.com  → 의도: [content]

  Task 1.2: 경쟁사 분석
     레퍼런스:
     > https://www.coupang.com/np/search?q=반려동물  → 의도: [competitor]
     > /Users/ceo/template.xlsx  → 의도: [template]

─────────────────────────────────────────────────────────────
프로젝트 3: 상품 등록

  Task 3.1: 제품 상세 페이지 디자인
     레퍼런스:
     > https://www.coupang.com/vp/products/123  → 의도: [style]
     > "미니멀하고 깔끔하게, 흰 배경에 제품 사진 크게"  → 의도: [note]
─────────────────────────────────────────────────────────────
```

**레퍼런스 타입:**
- URL: 웹 주소 (경쟁사, 참고 디자인 등)
- 이미지: 이미지 파일 경로 (목업, 스크린샷 등)
- 파일: 로컬 파일 경로 (보고서 템플릿, 데이터셋 등)
- 메모: 텍스트 메모 (스타일 가이드, 톤앤매너 등)

**레퍼런스 의도(intent):**
- `style`: "이런 스타일로 만들어줘"
- `content`: "이 내용을 참고해줘"
- `template`: "이 형식을 따라줘"
- `data`: "이 데이터를 활용해줘"
- `competitor`: "이 경쟁사를 분석해줘"

저장: `company/state/task_references.json`

### Step 4: Tool Agent 레퍼런스 분석 + Tool 추천

Tool Agent가 레퍼런스를 분석하여 최적의 Tool을 추천합니다.

**분석 과정:**
1. 레퍼런스 타입 + 의도 분석
2. URL은 WebFetch로 실제 내용 확인
3. 필요 Tool 도출 + 가용성 검증
4. Tool 조합 전략 수립

**출력 예시:**
```
══════════════════════════════════════════════════════════════
   Tool Agent: 레퍼런스 기반 Tool 추천
══════════════════════════════════════════════════════════════

Task 1.1: 시장 트렌드 조사
   레퍼런스: 네이버 데이터랩 URL [content]
   추천 Tool: WebSearch + WebFetch
   이유: 데이터랩 데이터 추출 + 추가 트렌드 검색
   사용 가능 여부: 모두 available

Task 1.2: 경쟁사 분석
   레퍼런스: 쿠팡 검색 URL [competitor] + 엑셀 템플릿 [template]
   추천 Tool: WebSearch + WebFetch + Write
   이유: 쿠팡 데이터 수집 → 템플릿 형식으로 보고서 작성
   사용 가능 여부: 모두 available

Task 3.1: 제품 상세 페이지 디자인
   레퍼런스: 쿠팡 상품 페이지 [style] + "미니멀 디자인" [note]
   추천 Tool: /frontend-design + WebFetch
   이유: 레퍼런스 페이지 구조 분석 → 미니멀 스타일 HTML/CSS 생성
   사용 가능 여부: 모두 available
   조합 전략: WebFetch → 구조 분석, /frontend-design → 코드 생성
```

저장: `company/state/tool_recommendations.json`

### Step 5: CEO Tool 추천 승인

```
══════════════════════════════════════════════════════════════
   CEO: Tool 추천 승인
══════════════════════════════════════════════════════════════

Tool Agent의 추천을 확인하고 승인해주세요.
수정이 필요하면 해당 태스크의 Tool을 변경할 수 있습니다.

  Task 1.1: WebSearch + WebFetch        [승인] [수정]
  Task 1.2: WebSearch + WebFetch + Write [승인] [수정]
  Task 3.1: /frontend-design + WebFetch  [승인] [수정]

  [전체 승인] [선택 수정]
```

저장: `company/state/tool_selections.json`

### Step 6: HR 에이전트 고용

레퍼런스와 확정된 Tool 기반으로 Expert 에이전트를 고용합니다.

```
══════════════════════════════════════════════════════════════
   HR: 에이전트 고용 완료
══════════════════════════════════════════════════════════════

┌──────────────────────────────────────────────────────────────┐
│ Expert 1: 데이터 기반 트렌드 리서처                           │
├──────────────────────────────────────────────────────────────┤
│ 역할: 네이버 데이터랩 등 데이터 소스 기반 시장 분석           │
│ Tools: WebSearch, WebFetch                                   │
│ 레퍼런스: 네이버 데이터랩 URL, 쿠팡 카테고리 URL 참고         │
└──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│ Expert 2: 미니멀 UI 디자이너                                  │
├──────────────────────────────────────────────────────────────┤
│ 역할: 쿠팡 스타일 미니멀 상품 페이지 디자인                   │
│ Tools: /frontend-design, WebFetch                            │
│ 레퍼런스: 쿠팡 상품 페이지 URL, "미니멀 깔끔" 스타일 가이드   │
└──────────────────────────────────────────────────────────────┘
```

저장: `company/state/hired_agents.json`

### Step 7: RM 태스크 할당

RM이 각 태스크에 적합한 에이전트를 할당합니다.

저장: `company/state/task_assignments.json`

### Step 8: CEO 프로젝트 선택

```
══════════════════════════════════════════════════════════════
   실행 준비 완료
══════════════════════════════════════════════════════════════

실행할 프로젝트를 선택해주세요:

  [1] 쿠팡 입점 준비 (3 tasks)
  [2] 제품 소싱 (2 tasks)
  [3] 상품 등록 (3 tasks)
  [A] 모든 프로젝트 순차 실행

선택:
```

### Step 9: Expert 태스크 실행 (레퍼런스 참조)

선택된 프로젝트의 태스크를 **레퍼런스를 참조하며** 실행합니다.

```
══════════════════════════════════════════════════════════════
   실행 중: 프로젝트 3 - 상품 등록
══════════════════════════════════════════════════════════════

Task 3.1: 제품 상세 페이지 디자인
   담당: 미니멀 UI 디자이너
   Tool: /frontend-design, WebFetch

   [레퍼런스 확인]
   - 쿠팡 상품 페이지 구조 분석 중... (WebFetch)
     → 헤더 + 대형 이미지 + 스펙표 + 리뷰 레이아웃 확인
   - 스타일 가이드: "미니멀, 깔끔, 흰 배경, 제품 사진 크게"

   [실행]
   - /frontend-design 실행: 쿠팡 스타일 레이아웃 + 미니멀 디자인 반영
   - 결과: company/outputs/product_page_task006.html

   완료
```

저장: `company/state/execution_log.json`

### Step 10: 결과 보고

```
상태: 완료

결과물:
  1. 제품 상세 페이지 (미니멀 스타일, 쿠팡 레이아웃 참고)

  2. ...

다음 단계 제안:
  → 다음 프로젝트 실행 권장

계속하시겠습니까? [다음 프로젝트] [종료]
```

## 상태 파일 구조

```
company/state/
├── session.json              # 세션 정보 (phase, created_at)
├── ceo_goal.json             # CEO 목표
├── backlog.json              # RM 생성 백로그
├── task_references.json      # CEO 레퍼런스 (v3 신규)
├── tool_recommendations.json # Tool Agent 추천 결과 (v3 신규)
├── tool_selections.json      # CEO 승인된 최종 Tool
├── hired_agents.json         # HR 고용 에이전트
├── task_assignments.json     # 태스크 할당 정보
└── execution_log.json        # 실행 로그
```

## 사용 가능한 Tool 목록

### Built-in (항상 사용 가능)
- `WebSearch` - 웹 검색
- `WebFetch` - 웹 페이지 내용 가져오기
- `Read` - 파일 읽기
- `Write` - 파일 쓰기
- `Edit` - 파일 편집
- `Bash` - 명령어 실행
- `Glob` - 파일 검색
- `Grep` - 텍스트 검색

### Skills (설치된 경우)
- `/frontend-design` - 웹 UI 코드 생성
- `/mcp-builder` - MCP 서버 생성
- 기타 `.claude/skills/` 디렉토리의 스킬들

### MCP (설정된 경우)
- `figma` - Figma 디자인 연동
- `rube` - 워크플로우 자동화 (500+ 외부 앱)
- `github` - GitHub 연동
- 기타 설정된 MCP 서버들

## 주의사항

1. 이 스킬은 **Claude Code 환경**에서만 동작합니다
2. MCP 서버 사용 시 사전 설정이 필요합니다
3. 상태 파일은 `company/state/` 폴더에 JSON으로 저장됩니다
4. 세션 종료 후에도 상태가 유지되어 계속 진행 가능합니다
5. 레퍼런스는 선택사항이며, 없는 태스크는 Tool Agent가 태스크 유형 기반으로 추천합니다
