## 🛠 오늘 진행한 활동

### 1. 최종 고도화 기능 정의

최종 발표 전까지 반드시 구현해야 할 핵심 기능을 다음과 같이 정리함:

#### (1) 실제 에이전트 수행 로직 구현

* 단순히 역할과 프롬프트를 정의하는 것이 아니라,

  * 실제 실행 단계까지 이어지는 **수행 로직 필요성 재확인**
* n8n의 로컬 워크플로우 실행 방식을 참고하여

  * 트리거 → 실행 → 결과 반환의 구조를
    우리 에이전트 구조에 적용하는 방안 검토
* 목표:

  * 계획 수립이 아닌
  * **행동을 유발하는 실행 흐름 확보**

---

#### (2) HR Agent 구조 고도화

기존 구조:

* HR Agent가 `hired_agents.json` 파일만 생성
* 실제 에이전트는 생성되지 않음
* 즉, “조직도 문서화”에 머무름

개선 목표:

* HR Agent가 단순 JSON 생성이 아니라

  * 실제 Sub-Agent를 생성
  * 독립적으로 Task를 수행하도록 구조 변경
* 즉,
  **“조직 구성”이 아니라 “실제 조직 작동” 단계로 전환**

이는 AI Company 구조가
“개념적 조직”을 넘어
**실행 가능한 멀티 에이전트 시스템으로 발전하는 핵심 단계**로 정의함

---

### 2. Claude Code 및 유사 구조 리서치 방향 설정

* 현재 우리가 설계 중인 구조가
  **Claude Code와 매우 유사한 방향성**을 가지고 있음을 인지
* 이에 따라,

  * Claude Code의 구현 방식
  * OpenClode(OpenClaude 등 유사 프로젝트)
    의 구조를 분석해보자는 의견 도출

핵심 질문:

* Claude Code는

  * Tool을 어떻게 호출하는가?
  * Sub-process를 어떻게 관리하는가?
  * 상태(State)는 어떻게 유지하는가?
* 우리는

  * 어디까지 유사하고
  * 어디에서 차별화할 것인가?

---

### 3. 신기술 설명회 내용 공유

#### (1) Claude의 “팀메이트(Teammate)” 개념

* 단일 모델 호출이 아니라

  * 협업 구조를 가진 AI 구성 방식
* 우리 구조의 HR Agent + Sub-Agent 모델과 유사한 측면 존재
* 차이점:

  * 우리는 조직 생성 메타 레이어를 강조
  * Claude는 실행 보조 중심

---

#### (2) tmux (터미널 분할)

* 여러 프로세스를 동시에 실행·관리할 수 있는 구조
* 멀티 에이전트 구조 실험 시

  * 각 에이전트를 분리 실행하는 방식에 참고 가능
* 실행형 AI Company 구조에서
  **프로세스 분리 전략에 대한 아이디어 제공**

---

### 4. 최종 발표 전략 정리

#### (1) 데모 영상 준비

* 실제 실행 장면을 녹화하여

  * 배속 편집 후 발표 자료에 포함 예정
* 단순 개념 설명이 아니라

  * “동작하는 모습”을 보여주는 것이 핵심

---

#### (2) n8n과의 차별점 명확화 필요

논의 결과:

* 실행 흐름 자체는 n8n과 유사해질 가능성 존재
* 그러나 우리의 핵심 차이점은 다음과 같음

| 구분    | n8n    | 우리의 구조       |
| ----- | ------ | ------------ |
| 워크플로우 | 사람이 설계 | AI가 백로그 생성   |
| 실행 단위 | 노드 기반  | 에이전트 기반      |
| 조직 개념 | 없음     | HR Agent가 고용 |
| 확장 구조 | 수동 구성  | 동적 조직 생성     |

핵심 차별점:

* ✔️ **백로그 자동 생성**
* ✔️ **에이전트 고용 및 조직 구성 개념**
* ✔️ 목표 기반 조직 동적 생성

→ 단순 자동화 툴이 아니라
**“조직을 만드는 자동화 시스템”이라는 점을 강조하기로 합의**

---

* 데모 영상 제작
* 인스타그램 자동 업로드 기능 구현

특히,
**인스타 자동 업로드는 “실제 행동하는 AI Company”의 상징적 기능**으로 설정함
