## 🛠 오늘 진행한 활동

### 1. 에이전트 실행 구조 초기 구현

지금까지는 에이전트의 역할과 프롬프트 정의 중심으로 시스템을 설계해 왔으나,
이번 주차에서는 실제로 작업을 수행하는 **실행 구조 구현**에 집중함.

기본 실행 흐름은 다음과 같이 구성함.

```
목표 입력
    ↓
CEO Agent : 프로젝트 목표 분석
    ↓
HR Agent : 필요한 Expert Agent 정의
    ↓
Expert Agent : 작업 수행
    ↓
결과 생성
```

이를 통해 AI Company 구조가 단순한 개념 설계가 아니라
실제 작업을 수행하는 **실행 시스템**으로 동작하도록 구조를 구성함.

---

### 2. 외부 Tool 연동 방식 검토

AI Company 시스템이 실제 작업을 수행하기 위해 외부 Tool을 활용하는 구조를 검토함.

**검토한 Tool 예시**

| Tool | 용도 |
|------|------|
| Gemini | 이미지 생성 |
| HTML 기반 | 이미지 생성 |
| Python Pillow | 이미지 생성 |

또한 **Claude Code의 Tool 사용 방식**도 분석함.

Claude는 아래 세 가지를 함께 전송하여 모델이 적절한 Tool을 판단하여 호출하도록 설계되어 있음.

- 시스템 프롬프트
- 사용자 프롬프트
- Tool 선택 기준

이를 참고하여 우리 시스템에서도 **Agent가 상황에 맞는 Tool을 선택**하도록 하는 구조를 실험함.

---

### 3. 인스타그램 자동 업로드 기능 구현

AI Company 시스템이 실제 외부 플랫폼에서 행동하는 모습을 보여주기 위해
**인스타그램 자동 업로드 기능**을 구현함.

작업 흐름은 다음과 같이 구성함.

```
콘텐츠 생성
    ↓
이미지 생성
    ↓
캡션 작성
    ↓
인스타그램 업로드
```

이를 통해 단순히 콘텐츠를 생성하는 수준이 아니라
실제 SNS 플랫폼에 콘텐츠를 게시하는 **자동화 작업**까지 수행할 수 있도록 구현함.

> 이 기능은 최종 발표 시 **"실제로 행동하는 AI Company"** 를 보여주는 핵심 데모 요소로 활용할 예정임.

---

### 4. 병렬 실행 가능 구조 검토

프로젝트 실행 과정에서 작업 간 의존성이 없는 경우 **동시에 실행할 수 있는 구조**를 검토함.

**예시 — 아래 세 작업은 서로 의존성이 없어 병렬 실행 가능**

```
콘텐츠 작성  |  이미지 생성  |  해시태그 생성
     ↓               ↓               ↓
              (병렬 실행)
```

이를 통해 전체 작업 수행 시간을 줄이고
에이전트 기반 시스템의 **효율적인 작업 처리 구조**를 확보할 수 있음.
