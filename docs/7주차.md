## 🛠 오늘 진행한 활동

### 1. HR Agent 기반 Sub-Agent 생성 구조 구현

기존 시스템에서는 HR Agent가 단순히 `hired_agents.json` 파일을 생성하는 방식으로
조직 구조를 기록하는 수준에 머물러 있었음.

그러나 이 방식은 실제 에이전트 실행과 연결되지 않는 문제가 있어
다음과 같은 방향으로 구조를 개선함.

**개선 방향**

- HR Agent가 필요한 전문가 역할 정의
- 해당 역할에 맞는 Sub-Agent를 생성
- 각 Agent가 독립적으로 작업 수행

**구조**

```
CEO Agent
    ↓
HR Agent
    ↓
Sub-Agent 생성
    ↓
Task 수행
```

이를 통해 AI Company 시스템이 단순 조직 정의가 아닌
**실제로 작동하는 멀티 에이전트 구조**로 발전하도록 설계함.

---

### 2. 에이전트 팀(Agent Team) 구조 검토

Claude Code에서 제공하는 Agent Team 구조를 분석하여
우리 시스템에 적용 가능한 협업 모델을 검토함.

Agent Team 구조에서는 아래 방식을 사용하고 있음.

- 여러 에이전트가 협력하여 작업 수행
- 역할 기반으로 작업 분배
- 결과를 상호 검토하는 구조

이를 참고하여 우리 시스템에서도 **에이전트 팀 기반 협업 구조**를 적용하는 방향을 논의함.

---

### 3. 자가 피드백(Self-feedback) 구조 설계

AI Company 시스템이 결과 품질을 스스로 개선할 수 있도록
**자가 피드백 구조**를 설계함.

자가 피드백 방식은 다음 세 가지로 구성함.

#### (1) 단일 에이전트 자기 검토

```
결과 생성
    ↓
Self Review
    ↓
수정
```

Expert Agent가 자신의 결과를 검토하고 필요한 경우 스스로 수정하는 방식.

#### (2) 크로스 에이전트 리뷰

```
Expert A 결과
    ↓
Expert B 리뷰
    ↓
Expert A 수정
```

다른 에이전트가 결과를 검토하여 객관적인 피드백을 제공하도록 구성함.

#### (3) Orchestrator 품질 검증

```
결과 생성
    ↓
품질 평가
    ↓
기준 미달 → 수정
```

Orchestrator 또는 상위 Agent가 전체 결과 품질을 검증하는 구조를 설계함.
