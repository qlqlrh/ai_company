# PNU AI Booster 최종(중간)보고서

**프로젝트명**: ai_company - AI 에이전트로 구성된 회사
**작성일**: 2026-01-28
**버전**: v2.0

## 1. 프로젝트 소개

### 가. 배경 및 필요성

#### 1) 연구 배경

최근 AI 기술의 급격한 발전으로 인해 **Large Language Model(LLM)** 기반의 AI 에이전트가 단순한 질의응답을 넘어 **자율적인 의사결정과 실행 능력**을 갖추게 되었습니다. 그러나 현재 대부분의 AI 활용은 여전히 **단일 에이전트 기반의 도구적 활용**에 머물러 있으며, 복수의 AI 에이전트가 **조직적으로 협업**하여 복잡한 비즈니스 목표를 달성하는 시스템에 대한 연구는 초기 단계에 있습니다.

#### 2) 문제 인식

| 구분 | 기존 AI 활용 방식 | 문제점 |
|------|------------------|--------|
| 역할 | 단일 보조 도구 | 복잡한 프로젝트에서 일관성 부족 |
| 협업 | 인간이 직접 조율 | 확장성 한계, 인적 리소스 소모 |
| 실행 | 수동 지시 필요 | 자율적 판단 및 연속 실행 불가 |
| 도구 사용 | 고정된 도구만 사용 | 상황에 맞는 동적 도구 선택 불가 |

#### 3) 필요성

- **조직적 AI 협업 시스템**: 인간 조직처럼 역할을 분담하고 협업하는 AI 시스템 필요
- **자율적 목표 달성**: CEO가 목표만 제시하면 AI들이 자율적으로 계획하고 실행하는 체계
- **Human-in-the-Loop**: 핵심 의사결정에는 인간(CEO)이 개입하되, 실무는 AI가 자동 처리
- **Tool-Aware 워크플로우**: 사용 가능한 도구를 사전에 검증하고 최적의 도구를 선택하는 지능형 시스템

### 나. 개발목표 및 주요내용

#### 1) 개발 목표

> **"CEO가 목표를 정의하면, AI 에이전트들이 자율적으로 협업하여 목표를 달성하는 Agentic AI 시스템"**

본 프로젝트는 AI를 단순한 도구가 아닌 **조직의 구성원(직원)**으로 정의하고, 역할을 나누어 협업하며 하나의 회사를 운영하는 모델을 실험합니다.

#### 2) 핵심 가설

```
CEO(사용자)가 회사의 목표를 정의하면,
  → RM 에이전트가 프로젝트를 분해하고,
  → Tool Agent가 필요한 도구를 검증하며,
  → HR 에이전트가 전문가를 채용하고,
  → Expert 에이전트들이 실제 업무를 수행하여 목표를 달성할 수 있다.
```

#### 3) 주요 개발 내용

| 구분 | 내용 | 상태 |
|------|------|------|
| **에이전트 시스템** | CEO, RM, Tool, HR, Expert 5종 에이전트 설계 및 구현 | 완료 |
| **Tool-Aware 워크플로우** | CEO가 Tool을 선택하고 사전 검증하는 v2 워크플로우 | 완료 |
| **스킬 기반 실행** | Claude Code Skills로 각 에이전트 역할 구현 | 완료 |
| **상태 관리** | JSON 기반 세션 및 상태 파일 관리 | 완료 |
| **외부 Tool 통합** | MCP(Figma, Rube, GitHub 등), WebSearch 연동 | 완료 |

#### 4) AI Agent vs Agentic AI

본 프로젝트는 단순한 AI Agent가 아닌 **Agentic AI** 패러다임을 적용합니다:

| 구분 | AI Agent | Agentic AI (본 프로젝트) |
|------|----------|-------------------------|
| **판단** | 규칙 기반 | 목표 기반 자율 판단 |
| **행동** | 단일 태스크 | 다단계 추론 및 계획 |
| **도구** | 고정 도구 | 동적 선택 및 검증 |
| **소통** | 단방향 | 양방향 (Human-in-the-loop) |
| **학습** | 없음 | 컨텍스트 기반 적응 |

### 다. 사회적가치 도입 계획

#### 1) 노동 생산성 향상

- **반복 업무 자동화**: 시장 조사, 경쟁사 분석, 보고서 작성 등 정형화된 업무 자동 수행
- **의사결정 지원**: AI 에이전트가 데이터 기반 분석 결과를 제공하여 CEO의 의사결정 품질 향상
- **업무 효율성**: 24시간 중단 없이 업무 처리 가능

#### 2) 소상공인 및 스타트업 지원

- **진입 장벽 낮춤**: 인력이 부족한 소규모 사업자가 AI 에이전트를 통해 대기업 수준의 업무 수행 가능
- **비용 절감**: 초기 인력 채용 비용 없이 사업 운영 가능
- **예시 활용**: 이커머스 입점(쿠팡, 네이버 등), 마케팅, 재무 분석 등

#### 3) 교육적 가치

- **SDD(Story-Driven Development) 방법론 실험**: 유저 스토리 중심의 개발 프로세스 연구
- **멀티 에이전트 시스템 연구**: 복수 AI 에이전트 협업에 대한 학술적 기여
- **오픈소스 공개**: 연구 결과물을 공개하여 관련 분야 발전에 기여

#### 4) 기대 효과

```
┌─────────────────────────────────────────────────────────────┐
│                     기대 효과                                │
├─────────────────────────────────────────────────────────────┤
│  • 1인 기업의 10인 기업 수준 업무 수행 가능                    │
│  • 반복적 업무에서 해방되어 창의적 업무에 집중                  │
│  • AI-인간 협업의 새로운 패러다임 제시                        │
│  • 스타트업/소상공인의 경쟁력 강화                            │
└─────────────────────────────────────────────────────────────┘
```

#### 2) 에이전트 역할 분담

| 에이전트 | 역할 | 책임 |
|----------|------|------|
| **CEO** | 의사결정자 | 목표 정의, Tool 선택, 프로젝트 선택, 인터럽트 응답 |
| **RM** | 프로젝트 매니저 | 백로그 생성, 태스크 분해, 에이전트에 태스크 할당 |
| **Tool Agent** | 도구 검증자 | Tool 가용성 검증, 대안 제시, 마켓플레이스 검색 |
| **HR** | 에이전트 팩토리 | 전문가 에이전트 고용, Tool 배정 |
| **Expert** | 실무자 | 할당된 Tool로 태스크 수행, 결과 보고 |

## 3. 개발결과

### 가. 전체 시스템 흐름도

#### 1) 워크플로우 상세

| 단계 | 주체 | 설명 | 출력 |
|------|------|------|------|
| ① | CEO | 회사 목표/KPI/제약조건 입력 | `ceo_goal.json` |
| ② | RM | 목표를 프로젝트/태스크로 분해, Tool 카테고리 제안 | `backlog.json` |
| ③ | CEO | 각 태스크에 사용할 Tool 선택 | `tool_selections.json` |
| ④ | Tool Agent | Tool 가용성 검증, 대안 제시 | `tool_validations.json` |
| ⑤ | HR | 검증된 Tool 기반 Expert 에이전트 고용 | `hired_agents.json` |
| ⑥ | RM | 각 태스크에 적합한 에이전트 할당 | `task_assignments.json` |
| ⑦ | CEO | 실행할 프로젝트 선택 | - |
| ⑧ | Expert | 할당된 Tool로 태스크 실행 | `execution_log.json` |
| ⑨ | RM | 최종 결과 보고 | 결과 파일들 |

#### 2) 사용자 경험 예시

```
[CEO] 목표: 쿠팡 입점하고 월 매출 1000만원 달성

[RM] 백로그를 생성했습니다:
  📁 프로젝트 1: 쿠팡 입점 준비
    📋 Task 1.1: 시장 트렌드 조사
    📋 Task 1.2: 경쟁사 분석
    📋 Task 1.3: 제품 카테고리 선정
  📁 프로젝트 2: 제품 소싱
    📋 Task 2.1: 1688 제품 검색
    ...
  📁 프로젝트 3: 상품 등록
    📋 Task 3.1: 제품 상세 페이지 디자인
    ...

[System] 각 태스크에 사용할 Tool을 선택해주세요.

[CEO] Task 3.1에는 Figma 사용할게. Task 1.1은 웹검색.

[Tool Agent] Tool 검증 결과:
  ✅ Figma - 사용 가능
  ✅ 웹검색(Tavily) - 사용 가능
  ⚠️ Task 2.1 (1688 검색) - 전용 Tool 없음
    → 대안: 웹검색 + 수동 검색 조합 추천

[CEO] OK, 대안으로 진행

[HR] 에이전트를 고용했습니다:
  👤 트렌드 리서처 (Tools: tavily_search)
  👤 디자인 전문가 (Tools: figma)
  👤 이커머스 전문가 (Tools: google_sheets)

[RM] 태스크 할당 완료. 어떤 프로젝트부터 실행할까요?

[CEO] 1번부터 시작

[Expert] Task 1.1 시작: 시장 트렌드 조사 (using WebSearch)
...
```

### 나. 기능 설명

#### 1) CEO Agent

- **목표 입력**: 회사 목표, KPI, 제약조건 정의
- **Tool 선택**: 각 태스크에 사용할 도구 직접 지정
- **프로젝트 선택**: 실행할 프로젝트 우선순위 결정
- **인터럽트 응답**: 에이전트 요청에 대한 승인/정보 제공

#### 2) RM Agent (Resource Manager)

- **백로그 생성**: 목표를 프로젝트/태스크로 분해
- **태스크 할당**: 각 태스크에 적합한 에이전트 매칭
- **실행 오케스트레이션**: 태스크 실행 순서 관리

#### 3) Tool Agent

- **로컬 Tool 탐색**: Built-in, Skills, MCP 스캔
- **가용성 검증**: CEO가 선택한 Tool 사용 가능 여부 확인
- **대안 제시**: 사용 불가 시 유사 Tool 추천
- **마켓플레이스 검색**: 외부에서 필요한 Tool 검색

#### 4) HR Agent

- **역할 분석**: 목표 달성에 필요한 전문가 역할 도출
- **에이전트 정의**: 역할별 전문성, Tool 권한, 한계 정의
- **시스템 프롬프트 생성**: 각 Expert의 동작 방식 정의

#### 5) Expert Agent

- **태스크 실행**: 할당된 Tool로 업무 수행
- **인터럽트 생성**: 정보/승인 필요 시 CEO에게 요청
- **결과 보고**: 실행 결과 보고 및 로깅

## 4. 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| **Tool-Aware** | CEO가 Tool을 직접 선택하고, 사전에 검증받는 방식 |
| **Backlog** | RM이 목표를 분해한 프로젝트/태스크 목록 |
| **Tool Validation** | Tool Agent가 수행하는 가용성 검증 |
| **Interrupt** | Expert가 CEO에게 정보/승인을 요청하는 메커니즘 |
| **Phase** | 워크플로우의 현재 단계 (goal_input → execution → completed) |
| **Agentic AI** | 목표 기반 자율 추론과 도구 사용 능력을 갖춘 AI 시스템 |

## 5. v1 → v2 핵심 변경사항

| 항목 | v1 (기존) | v2 (신규) |
|------|----------|----------|
| 순서 | HR → RM → Expert | RM(백로그) → **Tool** → HR → RM(할당) → Expert |
| Tool 선택 | 시스템 자동 | **CEO가 직접 지정** |
| Tool 검증 | 실행 시점 | **계획 단계 (사전 검증)** |
| 기본 에이전트 | CEO, HR, RM | CEO, RM, **Tool Agent**, HR |
